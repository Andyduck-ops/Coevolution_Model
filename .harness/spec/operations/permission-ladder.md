# Permission Ladder & Structured Escalation

## 4 级权限阶梯（项目适配）

| 级别 | 能力范围 | 典型命令 | 升级触发 |
|---|---|---|---|
| L1 只读 | 读取代码/文档/参数 | `sed`, `cat`, `find`, `grep` | 需要执行验证命令时 |
| L2 验证 | 运行 lint/test/build | `ruff`, `pytest`, `compileall` | 需要改代码时 |
| L3 修改 | 编辑代码与文档 | `Edit/Write`, 生成脚本 | 需要外部系统交互时 |
| L4 外部 | 网络/仓库/发布操作 | `git push`, 远端 API | 必须人工确认边界 |

## 结构化升级（Structured Escalation）

当出现以下任一条件，必须升级并给出结构化求助：

1. 同类失败达到 2 次（第 3 次前必须换策略）。
2. 缺失关键上下文导致无法判断（例如论文参数定义不全）。
3. 需要执行 L4 操作（推送、发布、外部网络写入）。
4. 出现多方案不可自动裁决的权衡。
